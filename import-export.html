Import & export

Download the current entity as an Excel workbook, or load / restore data from a file. By default new rows on import are appended. For a full replace, use Full backup import below to clear existing products, customers, prescriptions, orders, and lots first.

CSV export by section

Download one CSV per area — same columns as each screen’s import / sample file (lighter than the full Excel export above).

When checked, all existing products, vendors, customers, prescriptions, sales orders, and purchase lots for this entity are removed before importing the workbook. Rows are then imported; if the file includes an Entity sheet with data, shop profile + invoice_format_json + terms are merged in after.

Use one file with optional sheets (each with a header row). The Entity sheet (one data row) carries shop name, address, reorder/alert levels, accepted_payments_json, invoice_format_json, and terms_and_conditions for round-trip. The CommonDetails sheet adds a json_snapshot row (full JSON) plus human-readable entity fields and a staff table — extended profile + staff merge after transactional data. Money columns use INR where noted. Import order: vendors → products → customers → Prescriptions + PrescriptionLines → Orders + OrderLines → lots (with LotLines); then the Entity row (if present), then CommonDetails (if present), then Staff (if present; staff in the Staff sheet overrides duplicate rows from older CommonDetails JSON).

Entity

1 row per export

Shop profile + invoice format state + terms. Merged into the current session on import (duplicate shop names in the same browser are rejected).

Staff

1 row each per person

Columns: id, name, email, phone, role (admin or staff), created_at, updated_at. Applied after CommonDetails; leave id blank for new users.

CommonDetails

JSON + table per export

Row json_snapshot holds entity profile JSON (same fields as the Entity sheet, for quick round-trip). Human-readable key/value below. Staff is on the Staff sheet only; older files may still include staff in JSON and those rows are merged too.

Products

— in database

Medicine master: name, code, pack, composition, etc.

Vendors

— in database

Suppliers: name, phone, GSTIN, address…

Customers

— in database

Patients / walk-ins: name, phone, email, address lines, city, state, pincode, notes — use this sheet to restore or bulk-add customers.

Prescriptions + PrescriptionLines

— prescriptions

Hospital-style scripts: header keyed by rx_key, patient by name/phone, plus one row per line (status, type, notes, secret notes). Requires customers in the DB or on the same file.

Orders + OrderLines

— sales orders

Order header (customer, date, discount, status) plus one row per line (product, qty, line total in INR, dosing flags). Requires customers and products in the same file or already in the DB.

Lots + LotLines

— purchase lots

Purchase lots and line items (products must exist or be on the same file).

The login screen uses the same Excel workbook format; from there, rows are merged into the session you open. On this page you can optionally use Full backup import to replace all entity data. Export from one browser and import on another to move data between devices.
